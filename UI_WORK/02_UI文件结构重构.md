# 02 - UI 文件结构重构（CSS 独立文件拆分）

**日期：** 2026-05-15
**修改范围：** CSS 文件结构
**状态：** ✅ 完成
** commit：** `ed2b6d9`

---

### 改动原因

- `index.css` 达 3000+ 行，所有组件和页面的样式全部堆在一起
- 多人协作时，同时改动 index.css 容易产生冲突
- FAB 按钮和浮窗面板的样式已经相对独立，适合先拆分出来
- 后续 UI 改动会越来越多，集中在单一文件里会越来越难维护

---

### 改动内容

#### 1. 新增文件
- `src/components/FAB.css` — 浮窗按钮专属样式（164行）
  - `.ai-fab-wrap`、`.ai-fab`、`.ai-fab-icon`、`.ai-fab-label`、`.ai-fab-ai`、`.ai-fab-feedback`
  - 移动端媒体查询内的 FAB 相关样式
- `src/components/Panel.css` — 浮窗面板共享样式（220行）
  - `.ai-panel`、`.ai-panel-header`、`.ai-panel-title`、`.ai-panel-close`
  - `.ai-panel-search`、`.ai-panel-input`、`.ai-panel-btn`
  - `.ai-panel-body`、`.ai-panel-item`、`.ai-score` 等

#### 2. index.css 变更
- 删除 FAB + 面板样式共 384 行
- 剩余 2678 行（全局样式 + 页面专属样式）
- 导入语句保持不变（App.tsx 仍全局引入 index.css）

#### 3. 组件 import 更新
- `AIAssistant.tsx` 新增：
  ```ts
  import "./FAB.css";
  import "./Panel.css";
  ```
- `Feedback.tsx` 新增：
  ```ts
  import "./FAB.css";
  import "./Panel.css";
  ```

#### 4. 检查脚本更新
- `scripts/check-css-classes.cjs`
  - CSS_FILES 数组新增 `FAB.css` 和 `Panel.css` 的扫描路径
  - 确保新增的 CSS 文件也在类名检查范围内
